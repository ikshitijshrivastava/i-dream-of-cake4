🎂 I Dream of Cake — PWA Setup Guide

Follow these steps to make your dashboard a full Android app with login, cloud sync and multi-device support.

Step 1 — Create a Free Firebase Project

  1. Go to https://console.firebase.google.com
  2. Click "Add Project" → Name it i-dream-of-cake
  3. Disable Google Analytics (not needed) → Click Create Project

Step 2 — Enable Authentication

  1. In Firebase Console → Click Authentication (left sidebar)
  2. Click "Get Started"
  3. Click "Email/Password" → Enable it → Save

Step 3 — Create Firestore Database

  1. In Firebase Console → Click Firestore Database
  2. Click "Create database"
  3. Choose "Start in production mode" → Next
  4. Select your region (choose asia-south1 for India) → Done

Step 4 — Set Security Rules

  1. In Firestore → Click Rules tab
  2. Replace all content with:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
  • Click Publish

    • Step 5 — Get Your App Config

    1. In Firebase Console → Click the ⚙️ gear icon → Project Settings
    2. Scroll down to "Your apps" section
    3. Click the </> (Web) icon → Register app → name it idoc-web
    4. Copy the firebaseConfig object shown

    Step 6 — Paste Config into index.html

    Important

    Open index.html in any text editor. Find this section (around line 300):

    const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
};

    Replace with the config you copied from Firebase. Save the file.

    Step 7 — Host the App (Free)

    Option A — Firebase Hosting (Recommended)
    1. Install Node.js from nodejs.org
    2. Open terminal/command prompt in the idoc-pwa folder
    3. Run these commands one by one:
    npm install -g firebase-tools
firebase login
firebase init hosting
  → Select your project
  → Public directory: . (just a dot)
  → Single-page app: Yes
  → Overwrite index.html: No
firebase deploy

    Your app will be live at https://YOUR-PROJECT-ID.web.app

    Option B — Netlify (Even Simpler)
    1. Go to https://netlify.com → Sign up free
    2. Drag and drop the entire idoc-pwa folder onto the Netlify dashboard
    3. Your app goes live instantly with a URL like https://random-name.netlify.app

    Step 8 — Install as Android App

    1. Open the hosted URL in Chrome on your Lenovo P12 or Samsung S24 Ultra
    2. Chrome will show a banner: "Add to Home Screen" — tap it
    3. Or tap the 3-dot menu → "Add to Home Screen" → "Install"
    4. The app icon appears on your home screen — it opens like a native app!
    5. Do the same on your second device — they share the same data automatically

    How Multi-Device Sync Works

    Real-time, Progressive
    • Every order is saved to Firebase cloud the moment you tap "Save Order"
    • The other device sees the new order within seconds automatically
    • Progressive updates — only changed orders are synced, existing ones are never overwritten
    • If you're offline, data saves locally and syncs when connection returns
    • Each user account has completely separate data

    Files in this package

    Firebase Spark (free) plan includes: 1GB storage, 50,000 reads/day, 20,000 writes/day, 1GB bandwidth. For a cake order business this is more than enough — you'd need thousands of orders per day to exceed limits.
